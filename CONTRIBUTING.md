# Contributing

Thanks for your interest in improving MotionPhotoWorkbench.

## Before Opening a Change

- Check whether the issue is reproducible on the current `master` branch
- Keep changes focused and easy to review
- Avoid mixing refactors and behavior changes unless they are tightly related

## Development Notes

- The app currently targets Windows with WinForms
- FFmpeg is required at runtime but is not stored in the repository
- Do not commit local caches, build outputs, IDE state, or personal media files

## Pull Requests

- Describe the user-facing impact clearly
- Mention any manual test steps
- Call out known limitations or follow-up work
- Prefer small PRs over large unrelated batches

## Issue Reports

Helpful issue reports usually include:

- steps to reproduce
- expected behavior
- actual behavior
- sample input characteristics, if relevant
- screenshots or error messages when available
